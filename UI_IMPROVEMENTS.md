# 問卷編輯頁面 UI/UX 改進總結

## 改進概覽

成功改進了問卷編輯頁面中的「顯示模式」和「問卷設定」區域的 UI 設計，提升了視覺清晰度、可讀性和使用體驗。

## 主要改進項目

### 1. 問卷描述區塊
**改進前：**
- 普通白色背景卡片
- 視覺層次不明顯
- 缺乏引導性

**改進後：**
- 使用藍色到靛藍色漸變背景（from-blue-50 to-indigo-50）
- 半透明白色輸入框（bg-white/80）支援毛玻璃效果
- 更友善的 placeholder 文字引導
- focus 狀態增強（ring-2 ring-blue-500）

### 2. 顯示模式選擇
**改進前：**
- 使用傳統 radio button
- 文字說明在下方，不夠直觀
- 選項之間缺乏視覺區分

**改進後：**
- **卡片式選擇器設計**
  - 每個模式獨立卡片展示
  - 配備專屬圖標和詳細說明
  - 選中狀態：藍色邊框 + 藍色背景 + 勾選標記
  - hover 效果增強交互反饋
- **視覺層次清晰**
  - 使用不同顏色區分選中/未選中狀態
  - 圖標使用統一尺寸（40x40px）的彩色背景容器

### 3. 問卷設定區塊
**改進前：**
- 簡單的 checkbox 列表
- 設定項之間缺乏分組
- 視覺密度過高

**改進後：**
- **卡片化設計**
  - 每個設定項獨立卡片包裝
  - hover 時顯示陰影效果
  - 使用網格佈局（grid）優化空間
- **圖標系統優化**
  - 每個功能配備彩色圖標容器
  - 自動跳題：靛藍色（indigo）
  - 返回上題：琥珀色（amber）
  - 顯示進度：綠色（green）
- **Toggle 開關美化**
  - 自定義 toggle 樣式
  - focus 狀態顯示藍色光環
  - 平滑的過渡動畫
- **關聯功能視覺化**
  - 延遲時間滑桿與自動跳題整合在同一卡片
  - 條件顯示邏輯（僅在開啟自動跳題時顯示）
  - 實時顯示當前延遲值（藍色標籤）

### 4. 間距與呼吸感
**改進前：**
- 區塊間距較小（mb-6）
- 內部元素緊湊

**改進後：**
- 區塊間距增大（mb-8）
- 卡片內邊距優化（p-5）
- 元素間距更合理（gap-3, gap-4）

### 5. 顏色系統優化
- **主色調**：藍色系（blue-500/600）用於主要操作
- **輔助色彩**：
  - 靛藍色：自動跳題功能
  - 琥珀色：返回功能
  - 綠色：進度顯示
- **背景層次**：白色 → gray-50 → 漸變色
- **狀態反饋**：選中狀態使用主題色，hover 狀態加深

## 技術實作細節

### CSS 類別使用
- Tailwind CSS utility classes
- 響應式設計（md:grid-cols-2）
- 過渡動畫（transition-all, transition-shadow）
- 自定義 toggle 開關樣式

### Vue 綁定優化
- 動態類綁定提升性能
- 條件渲染優化（v-if 用於選中標記）
- 事件處理保持原有邏輯不變

## 使用體驗提升

1. **更清晰的視覺層次**：通過顏色、大小、間距建立明確的信息層級
2. **更好的可點擊性**：卡片式設計增大點擊區域，hover 效果提供即時反饋
3. **功能分組明確**：相關設定視覺上聚合，提升理解效率
4. **降低認知負荷**：圖標、顏色、說明文字協同工作，減少理解成本
5. **視覺愉悅度提升**：漸變背景、圓角設計、平滑動畫提升整體美感

## 響應式設計
- 移動端：單列布局
- 平板及以上：雙列網格布局
- 所有交互元素都考慮了觸控友好性

## 無障礙性考量
- 保留語義化 HTML 結構
- Toggle 開關使用 sr-only 類隱藏原生 checkbox
- 適當的 label 關聯
- 顏色對比度符合 WCAG 標準
